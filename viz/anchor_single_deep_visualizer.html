One anchor, read deep

The single-sequence digit structure of one bulk anchor M(ω), ω = 4996160569905494617, computed to 217 = 131,072 two-adic bits — 32× deeper than the 4,096-bit breadth battery (anchors.md §17.7.1). Every panel plots the anchor (blue) against a matched i.i.d. fair-coin control (grey) and the exact fair-coin null (orange). The question is whether one number's own digit string, read as far as we can afford, departs from a coin — the single-sequence formalization of the equidistribution AEH conjectures. Companion to Reduced coordinates for the Collatz map, DOI 10.5281/zenodo.21273548.

1 · Run-length distribution

Counts of maximal runs of identical bits. Fair-coin null: P(L=k)=2−k — geometric decay, not a bell curve. Log scale.

2 · Window bit-sum — the "bell curve"

Number of 1s in each of 512 non-overlapping 256-bit windows. Fair-coin null: Binomial(256,½) ≈ Normal(128, 8²) — this is a bell curve. (Necessary, not sufficient: the CLT makes almost any bit-sum look Gaussian even under correlation — panels 1, 3 are the discriminating tests.)

3 · Block-entropy scaling

Empirical entropy Hk of overlapping k-bit blocks, plotted as Hk/k. Fair-coin null: Hk=k, so Hk/k=1. The droop past k≈10 is finite-sample undercounting — the control droops identically, so only the M−control gap (near zero here) would signal real compressibility.

Verdict (anchors.md §17.7.2): a clean pass on this bulk anchor across run-length, window bit-sum, autocorrelation (max |z| = 2.78 over lags 1–512, below the Bonferroni bar 4.27), block entropy, and zlib — all matching the control. Deep single-sequence calibration; not a statement about AEH, which concerns the measure-zero set of integer orbits (see §17.8 item 5).

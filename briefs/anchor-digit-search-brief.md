# Brief: single-sequence structure search on the anchor `M(ω)` (for a delegated session)

**Context required before starting (in order):** `README.md` (esp. the strategy/stopping rules), `AGENTS.md` (binding house norms), `anchors.md` §17 in full (the consolidated reference this brief extends — §17.7–17.9 are the scope and plan you are executing), `aeh.md` §13 (the existing cross-sectional statistics — know what's already been checked so you don't repeat it), `archive/appendix-a.md` A.4.6 (historical precedent: digit-density counts, and the refuted `ω mod 3` correlate — a closed door, don't reopen it without new reason).

## The state you are starting from

The anchor `M(ω)` is an infinite 2-adic bit string attached to every odd `ω` (definition: `anchors.md` §17.1). Everything proved about it so far is either an exact per-step law (§17.3) or a cross-sectional statistic — a frequency or correlation averaged over *many* different `ω` or orbit visits (aeh.md §13, `anchors.md` §17.6). Nothing in the wiki has ever tested whether *one* `M(ω)`, read as a single bit string, has internal structure: autocorrelation, periodicity, compressibility, self-similarity. That is confirmed, not assumed — every experiment script in `experiments/` was checked (2026-07-12) and none does this.

This is explicitly in scope under the stopping rules (README): AEH *proof* effort is parked, but "experiments may feed the ledger." This brief is that kind of experiment — not an attempt to prove or disprove AEH, a structure hunt that either finds something (real progress) or comes back clean (a legitimate, recordable strengthening of the calibration on a new axis).

## Task

Run the search plan in `anchors.md` §17.8, build the two priority visualizations in §17.9, and report findings — including a clean pass — with the same rigor the rest of this wiki applies to computational claims.

## Queue, in order (each stage stands alone; a clean negative at any stage is a valid, recordable outcome — do not treat "no structure found" as failure)

1. **Single-sequence statistics** (§17.8.1): run-length distribution vs. the geometric null, autocorrelation at lags `1..~64`, chi-squared on 3–5-bit overlapping windows — each against a real number of independently generated `M(ω)` strings (suggest: at least 200 values of `ω`, each to at least 4,096 bits, generated by the congruence definition directly — not copied from any existing script's output). State the null model precisely (fair coin, independent bits) and the test statistic's distribution under that null before reporting a deviation.
2. **Compressibility** (§17.8.2): Lempel–Ziv complexity or a standard compressor on the raw bit strings; separately, run the NIST SP 800-22 battery if a usable implementation is available (do not hand-roll a replacement for tests that already exist and are well-calibrated).
3. **Visualizations** (§17.9, first tier): build the recurrence plot and the chaos-game representation for a handful of representative `ω` (include at least one from the "bottom" — small `ω` — and several from the bulk, to visually confirm the known bottom/bulk distinction shows up before looking for anything new in the bulk). Interactive piece goes in `viz/`, following the existing convention (`anchor_field_explorer.html`).
4. **If time remains:** spectral test (§17.8.3), the cheap cross-string correlate sweep (§17.8.4), second-tier visualizations (§17.9).
5. **Out of scope for this brief:** the Bernstein–Lagarias literature read (§17.8 item 5) is a separate, non-computational task — note it as a candidate follow-up, do not attempt to resolve it here.

## Explicit success / stop criterion

- **A discovered structure** (any test rejects the coin-flip null at a real significance level, replicated on an independent fresh sample): do not chase it into a claim about AEH's truth or falsity. Report exactly what was found, at what scale, with what statistical confidence, and stop to let it be reviewed before deciding what it means. This would be a genuine discovery — treat it as carefully as the KL–LP brief's precedent treats an unexpectedly good number (verify before believing).
- **A clean pass** (every test run is consistent with the coin-flip null, at stated significance levels, over the stated sample): this is a valid, complete, recordable outcome. Write it up as such — which tests, what parameters, what range, zero rejections — the same way a "zero failures" verification line reads elsewhere in this wiki. Do not keep grinding for a positive result once the planned battery is exhausted; report the negative and stop.
- Either way, this brief does not conclude "AEH is proved" or "AEH is refuted." It concludes "single-sequence structure was searched for, here's what was and wasn't found."

## Rules (non-negotiable, from AGENTS.md)

- Every claim — positive or a clean pass — gets fresh, independently-written code (not copied from `aeh_calibration.py` or `digit_duality.py`, even though those compute related quantities). Record exactly what was checked: sample size, bit depth, seeds if random, date.
- Distinguish rigor levels explicitly: this is empirical/computational, not proof. Never phrase a finding as if it settles AEH.
- Failures and dead ends are recorded, not deleted — if a test implementation turns out wrong partway through (cf. the KL–LP brief's recorded refuted constructions), fix it, note the correction, keep both in the record.
- **Numbering:** findings update `anchors.md` §17.7 in place (new subsections `17.7.x` as needed); do not open a new page.
- Work on branch **`anchor-digits`**, commit per stage with verification in the same commit, and **do not merge to main** — the main session reviews and re-runs all verification code before merging (mirror-queue precedent).
- Code goes in `experiments/` (suggested `experiments/anchor_digit_structure.py` for the statistical battery, `experiments/anchor_visualize.py` or a notebook feeding `viz/anchor_digit_visualizer.html` for the plots); each states which result it supports and stays runnable.
- No scope expansion. Anything interesting-but-off-brief goes in `briefs/anchor-digit-search-findings.md`, not into the main line.

## Definition of done

Either (a) a documented structure, with the exact test, statistic, significance level, and sample that produced it, flagged clearly as unreviewed pending main-session verification; or (b) a documented clean pass across the planned battery, with exact parameters and ranges — written into `anchors.md` §17.7, verification recorded in `AGENTS.md`, branch `anchor-digits` with clean per-stage commits, visualizations committed to `viz/`. Either outcome ends with an honest one-paragraph status update to §17.7 and §17.10.
